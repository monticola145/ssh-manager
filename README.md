# SSH Manager TUI

Терминальное TUI‑приложение для удобного подключения к серверам по SSH с сохранёнными профилями.

## Возможности

- **Список серверов**: профили SSH с именем, host, port, user, типом аутентификации и описанием.
- **Быстрое подключение**: выбор профиля и запуск `ssh` без ручного ввода команды.
- **Поддержка SSH ключей и паролей** (для паролей желательно использовать SSH‑ключи или системный keyring).
- **Встроенный блокнот**: локальные текстовые заметки с автосохранением.
- **Вкладка Help**: горячие клавиши и краткая инструкция.
- **Горячие клавиши**:
  - `↑` / `↓` — выбор сервера
  - `Enter` — подключение
  - `a` — добавить профиль
  - `e` — редактировать профиль
  - `d` — удалить профиль
  - `/` — поиск
  - `Tab` — переключение вкладок
  - `Esc` — выход назад / закрытие сеанса
  - `Ctrl+H` — открыть help
  - `Ctrl+N` — открыть блокнот
  - `Ctrl+Shift+C` — копировать (в поддерживаемых виджетах)
  - `Ctrl+Shift+V` — вставить (в поддерживаемых виджетах)

## Хранение данных

По умолчанию данные хранятся в:

- `~/.ssh-manager/config.json`
- `~/.ssh-manager/profiles.json`
- `~/.ssh-manager/notes.txt`

Пароли для SSH‑подключений в MVP‑версии могут храниться в `profiles.json` в открытом виде.
Рекомендуется использовать SSH‑ключи (`auth_type = "ssh_key"`) и агент (`ssh-agent`) для безопасной аутентификации без пароля.

## Установка и запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Запуск приложения из корня проекта:

```bash
python main.py
```

## Запуск из любой директории

Есть несколько способов, как вызывать приложение, находясь в любой папке.

### Вариант 1: alias в bash (Git Bash / WSL)

Добавьте в `~/.bashrc` (или `~/.bash_profile`) строку:

```bash
alias sshm='python "c:/Dev/console/main.py"'
```

Затем перечитайте конфиг:

```bash
source ~/.bashrc
```

Теперь можно запускать менеджер профилей так:

```bash
sshm
```

### Вариант 2: `.bat`‑файл в `PATH` (для cmd/PowerShell)

1. Создайте файл `ssh-manager.bat` в папке, которая есть в переменной окружения `PATH`
   (или создайте свою папку и добавьте её в `PATH`).
2. Поместите в `ssh-manager.bat` следующий код:

```bat
@echo off
python "c:\Dev\console\main.py" %*
```

После этого из любой директории можно запускать:

```bash
ssh-manager
```

### Вариант 3: установка как Python‑пакета

Более «правильный» способ — оформить проект как пакет с entry point’ом `console_scripts`.
После этого можно будет сделать `pip install .`, и команда (например) `ssh-manager`
станет доступна глобально. На данный момент это не настроено, но проект можно
легко доработать под такой сценарий.
