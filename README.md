# SSH Manager

Консольное приложение для удобного подключения к серверам по SSH с сохранёнными профилями.

## Возможности

- **Список серверов**: профили SSH с именем, `host`, `port`, `user`, типом аутентификации и описанием.
- **Подключение по профилю**: выбор профиля и открытие SSH‑сессии через `paramiko`.
- **Добавление профилей**: интерактивные вопросы в консоли и сохранение в `profiles.json`.
- **Поддержка SSH‑ключей и паролей**: в профиле можно указать либо пароль, либо путь к приватному ключу (RSA, Ed25519, ECDSA). Если ключ защищён passphrase, его можно сохранить в профиле или ввести при подключении.
- **Несколько сессий и переключение по горячим клавишам**: можно держать несколько SSH-сессий и переключаться между ними. Работает в bash (WSL, Linux, macOS, Git Bash при наличии termios), а также в консоли Windows (cmd/PowerShell).

### Горячие клавиши (режим сессий)

| Клавиши   | Действие                          |
|-----------|-----------------------------------|
| **Ctrl+N** | Следующая сессия                  |
| **Ctrl+P** | Предыдущая сессия                 |
| **Ctrl+Q** | Выйти в главное меню (сессии живы) |
| **Ctrl+X** | Закрыть текущую сессию            |

Обычные символы (буквы, Enter, Backspace) уходят в текущую SSH-сессию.

- **Графический интерфейс (GUI)**: окно с списком профилей, кнопки «Подключиться», «Изменить», «Удалить», «Добавить профиль». Подключение открывается в отдельном окне с выводом и полем ввода. Запуск: `python gui.py`.

## Хранение данных

По умолчанию данные о профилях хранятся в файле:

- `profiles.json` — рядом с `main.py` (каталог проекта, например `c:/Dev/ssh-manager/profiles.json`).

Пароли для SSH‑подключений в текущей версии хранятся в `profiles.json` в открытом виде.
Рекомендуется использовать SSH‑ключи (`auth_type = "ssh_key"`) и агент (`ssh-agent`) для безопасной аутентификации без пароля.

## Установка и запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Запуск из корня проекта:

- **Консольное приложение** (меню в терминале, несколько сессий, горячие клавиши):

```bash
python main.py
```

- **Графический интерфейс** (окно с тёмной темой, список профилей, кнопки):

```bash
python gui.py
```

Профили общие: оба варианта используют один и тот же `profiles.json`.

## Запуск из любой директории

Есть несколько способов, как вызывать приложение, находясь в любой папке.

### Вариант 1: alias в bash (Git Bash / WSL)

Добавьте в `~/.bashrc` (или `~/.bash_profile`) строку:

```bash
alias sshm='python "c:/Dev/ssh-manager/main.py"'
```

Затем перечитайте конфиг:

```bash
source ~/.bashrc
```

Теперь можно запускать менеджер профилей так:

```bash
sshm
```

### Вариант 2: `.bat`‑файл в `PATH` (для cmd/PowerShell)

1. Создайте файл `ssh-manager.bat` в папке, которая есть в переменной окружения `PATH`
   (или создайте свою папку и добавьте её в `PATH`).
2. Поместите в `ssh-manager.bat` следующий код:

```bat
@echo off
python "c:\Dev\ssh-manager\main.py" %*
```

После этого из любой директории можно запускать:

```bash
ssh-manager
```

### Вариант 3: установка как Python‑пакета

Более «правильный» способ — оформить проект как пакет с entry point’ом `console_scripts`.
После этого можно будет сделать `pip install .`, и команда (например) `ssh-manager`
станет доступна глобально. На данный момент это не настроено, но проект можно
легко доработать под такой сценарий.

## Сборка в exe (Windows)

Упаковать приложение в один исполняемый файл можно с помощью [PyInstaller](https://pyinstaller.org/).

1. Установите PyInstaller:

```bash
pip install pyinstaller
```

2. В корне проекта выполните:

```bash
pyinstaller --onefile --name ssh-manager main.py
```

Готовый `ssh-manager.exe` будет в папке `dist/`. Файл `profiles.json` приложение ищет в той же папке, где лежит exe (и создаёт его при добавлении первого профиля).

Рекомендуется положить exe в отдельную папку (например, `C:\Tools\ssh-manager\`) и при необходимости скопировать туда свой `profiles.json`.
