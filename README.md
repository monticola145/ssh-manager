# SSH Manager

Консольное приложение для удобного подключения к серверам по SSH с сохранёнными профилями.

## Возможности

- **Список серверов**: профили SSH с именем, `host`, `port`, `user`, типом аутентификации и описанием.
- **Подключение по профилю**: выбор профиля и открытие SSH‑сессии через `paramiko`.
- **Добавление профилей**: интерактивные вопросы в консоли и сохранение в `profiles.json`.
- **Поддержка SSH‑ключей и паролей**: в профиле можно указать либо пароль, либо путь к приватному ключу.

## Хранение данных

По умолчанию данные о профилях хранятся в файле:

- `profiles.json` — рядом с `main.py` (каталог проекта, например `c:/Dev/ssh-manager/profiles.json`).

Пароли для SSH‑подключений в текущей версии хранятся в `profiles.json` в открытом виде.
Рекомендуется использовать SSH‑ключи (`auth_type = "ssh_key"`) и агент (`ssh-agent`) для безопасной аутентификации без пароля.

## Установка и запуск

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Запуск приложения из корня проекта:

```bash
python main.py
```

## Запуск из любой директории

Есть несколько способов, как вызывать приложение, находясь в любой папке.

### Вариант 1: alias в bash (Git Bash / WSL)

Добавьте в `~/.bashrc` (или `~/.bash_profile`) строку:

```bash
alias sshm='python "c:/Dev/ssh-manager/main.py"'
```

Затем перечитайте конфиг:

```bash
source ~/.bashrc
```

Теперь можно запускать менеджер профилей так:

```bash
sshm
```

### Вариант 2: `.bat`‑файл в `PATH` (для cmd/PowerShell)

1. Создайте файл `ssh-manager.bat` в папке, которая есть в переменной окружения `PATH`
   (или создайте свою папку и добавьте её в `PATH`).
2. Поместите в `ssh-manager.bat` следующий код:

```bat
@echo off
python "c:\Dev\ssh-manager\main.py" %*
```

После этого из любой директории можно запускать:

```bash
ssh-manager
```

### Вариант 3: установка как Python‑пакета

Более «правильный» способ — оформить проект как пакет с entry point’ом `console_scripts`.
После этого можно будет сделать `pip install .`, и команда (например) `ssh-manager`
станет доступна глобально. На данный момент это не настроено, но проект можно
легко доработать под такой сценарий.
